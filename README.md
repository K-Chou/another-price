# 另外的价钱

一个支持自定义主题（加班付费 / 跑腿费 / 份子钱 / AA 等）的轻量级收款 H5 应用，朋友点开链接即可扫码付你。

## 功能

- 个人主页：昵称、头像、一句话简介
- 加班价目表：自由新增/编辑/删除条目，每条包含 **图标 + 名称 + 时长 + 金额**
- 创建账单：勾选价目并调整数量、临时新增"时长 × 单价"项目、加备注、自动汇总
- 分享链接：一键复制链接发到微信，**对方无需登录、无需安装**即可查看
- 付款页：被分享人看到金额明细，长按你的微信/支付宝收款码识别付款
- 账单历史：所有创建过的账单都保留，可手动标记"已收款"

## 文件结构

```
.
├── index.html          # 入口 HTML
├── styles.css          # 样式
├── app.js              # 应用逻辑（SPA 路由 + 数据持久化）
├── feishu-lemon.html   # 视觉参考（可删）
└── README.md           # 当前文件
```

只有 3 个核心文件（`index.html` / `styles.css` / `app.js`），没有任何依赖，没有构建步骤。

## 本地预览

任意一个静态服务器即可。最简单：

```bash
# 在项目根目录执行（任选其一）
python3 -m http.server 5173
# 或者
npx serve -p 5173
```

然后浏览器打开 `http://localhost:5173`。

> ⚠️ 不能直接双击 `index.html` 打开（`file://` 协议下 Clipboard API 等部分功能会受限）。

## 数据存储说明

- 你的**头像、昵称、价目表、收款码、账单历史**都只保存在 **当前浏览器的 localStorage** 中。
- 收款码图片会被自动压缩后 base64 存到本地，不会上传到任何服务器。
- 分享链接里包含**对方付款所需的全部信息**（发起人昵称、头像、收款码、账单明细），所以链接会比较长。这是为了让对方在没有后端的情况下也能看到完整页面。
- 清空浏览器缓存 / 换设备 = 数据会丢失。如果需要多端同步、防止丢失，请阅读下文「升级到后端」。

## 三种部署方案

> 你在前面问过：**微信小程序审核严的话，能否部署到外部服务器**？答案是 **完全可以，而且本应用更适合走 H5 路线**。

### 方案 ① 静态托管到自有/云服务器（推荐，零成本）

适合场景：**99% 的用户首选**。无需备案（如果用境外服务）、无需小程序资质，分享链接发到微信即可使用。

#### 1.A 用 Vercel / Netlify / Cloudflare Pages（免费、免备案、全球 CDN）

以 **Vercel** 为例：

```bash
# 1) 在项目目录初始化 git
git init && git add . && git commit -m "init"

# 2) 推送到 GitHub（先在 GitHub 建个空仓库）
git remote add origin git@github.com:<你的用户名>/<仓库名>.git
git push -u origin main

# 3) 打开 https://vercel.com → Import GitHub repo → 一路 Next → Deploy
#    部署完成后会给你一个 https://xxx.vercel.app 的域名
```

**这个域名生成的分享链接，发到微信里可以直接打开**。

#### 1.B 用国内云服务器（腾讯云/阿里云）

如果想用自己的国内域名，需要先做 **ICP 备案**（约 10 天，免费）。备案后：

```bash
# 1) 把 index.html / styles.css / app.js 三个文件上传到服务器
scp index.html styles.css app.js root@<服务器IP>:/var/www/jiaban/

# 2) 配置 Nginx（最简配置）
server {
    listen 80;
    server_name jiaban.example.com;
    root /var/www/jiaban;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }
}

# 3) 用 certbot 配置 HTTPS（微信里打开必须 HTTPS）
sudo certbot --nginx -d jiaban.example.com
```

#### 微信内打开的注意事项

- **必须 HTTPS**：微信浏览器对 http:// 链接有强提示，部分功能会限制。
- **域名建议先备案**（如果用国内服务器）：未备案的国内域名会被微信拦截。
- **复制粘贴**：本应用的"复制"按钮在 HTTPS 下用 `navigator.clipboard`，HTTP 下会降级到 `execCommand('copy')`。
- **二维码识别**：微信内打开 H5 页面时，长按图片即可识别图中二维码并跳转付款，这是微信原生能力，**100% 可用，无需任何接入**。
- **分享卡片**：H5 在微信里没有原生卡片预览（需要公众号 JS-SDK），所以"复制按钮"已经把**账单标题 + 金额 + 明细 + 链接**打包成一段格式化文案，粘贴到聊天里同样体面。

#### 域名风险与应急切换

如果你大规模公开传播，**域名有可能被微信安全策略加红**（用户看到"非官方网页"提示）。这是 H5 + 微信的常态风险，但有解：

**预防**：
- 用 `vercel.app` / `pages.dev` / `netlify.app` 这类**官方子域**，整域已被微信认可，被加红概率显著降低
- 不要在文案里写"转账 / 收款 / 提现 / 代付"等敏感词（本仓库默认文案已避开）
- 避免大群、朋友圈强推；私域传播几乎零风险

**应急**：被加红后**几分钟即可换域名重新部署**：
1. 在 Vercel/Cloudflare Pages 上**重新 New Project**，选同一个 GitHub 仓库（或直接拖目录上传），会生成一个**全新的 vercel.app 子域**
2. 让朋友改用新链接（旧账单数据存在本地 localStorage 不会丢，新域名打开就能继续）
3. 如有自有域名，DNS 切到新部署即可

> 因为账单数据本来就在用户本地浏览器，**换域名不会丢任何东西**。

## PWA：把它装到手机主屏当 App 用

这是个完整的 PWA，对**重度用户**特别值得装一下：

- iOS Safari：打开页面 → 分享菜单 → **"添加到主屏幕"**
- 安卓 Chrome：打开页面 → 菜单 → **"安装应用"** / "添加到主屏"
- 微信内打开：先**右上角 → 在浏览器中打开**，再按上面的步骤

装上之后：
- 主屏多一个 App 图标（来自 `assets/logo-192.png`、`logo-512.png`）
- 启动是**全屏 standalone**，没有浏览器地址栏
- **首次访问后离线也能打开**（Service Worker 缓存了所有静态资源）
- 数据仍然完全在本地，不依赖网络

### 方案 ② 改造为微信小程序

适合场景：希望进入微信"小程序"入口、想用 `wx.getUserProfile` 获取真实头像昵称。

#### 资质要求

- **个人主体**：可以注册个人小程序，**但无法开通微信支付**。如果你只需要展示收款码，个人主体小程序**完全够用**。
- **企业/个体工商户**：可以注册认证小程序，可以开通微信支付（需要营业执照、年费 ¥300）。

#### 迁移工作量评估

本应用 **可以平移到小程序**，但需要改写视图层。原因：

| 当前实现 | 小程序对应 |
|---|---|
| `innerHTML` 渲染 HTML | 改用 `.wxml` 模板 + `setData` |
| CSS（`px`） | 改用 `.wxss`（`rpx` 自适应） |
| `localStorage` | 改用 `wx.setStorageSync` |
| URL hash 路由 | 改用 `wx.navigateTo` 多页面 |
| 分享链接 | 改用 `onShareAppMessage`（小程序卡片分享） |
| 上传二维码 | 改用 `wx.chooseImage` |
| 复制链接 | 改用 `wx.setClipboardData` |

**预计 2-3 天迁移工作量**。如果你确认要走小程序路线，告诉我，我可以再帮你生成一份小程序原生代码（`pages/` 目录结构）。

#### 小程序审核重点

- 类目选择：建议选 **"工具" → "效率"**，不要选金融/支付类。
- 描述话术：**不要出现"代收款""转账""提现""佣金"等敏感词**。可以说"加班时长记录与费用结算工具"。
- 服务说明：在「设置-基本设置」里写清楚 **"用户间的收款仅展示个人收款码，不涉及平台代收代付"**。
- 隐私协议：必须有，可以用[微信官方模板](https://developers.weixin.qq.com/community/develop/article/doc/000288ad568dd07e2a5be2fa056413)。
- 「不接入微信支付」**反而更容易过审**，因为不会触发"虚拟交易"等高风险审查。

## 常见问题

**Q: 分享链接为什么这么长？**
A: 因为链接里塞了发起人昵称、头像、二维码、账单明细，避免对方需要登录或后端调用。如果上后端，可以缩短为 `?id=xxx`。

**Q: 在微信里点开链接，长按二维码识别不出来？**
A: 这是发起人上传的**收款码图片本身**不清晰造成的。请重新从微信"收款" → "保存收款码"导出高清原图后上传。本应用会自动压缩到 600px，对正常的收款码足够。

**Q: 苹果手机微信里复制按钮没反应？**
A: 部分老版本微信 WebView 不支持 `navigator.clipboard`。本应用已经做了降级，仍然失败的话，可以长按链接区域手动选择复制。

**Q: 数据会丢吗？**
A: 在同一浏览器里只要不清缓存就不会丢。换手机/换浏览器**一定要先在原设备生成分享链接备份**，或者升级到后端方案。
